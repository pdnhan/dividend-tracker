# Codemaps Index

**Last Updated:** 2026-03-19

This directory contains architectural maps and documentation of the dividend tracker codebase structure.

## Available Codemaps

### [ARCHITECTURE.md](ARCHITECTURE.md)
High-level system overview showing component relationships, data flow diagrams, and external dependencies. Provides a architectural perspective of how the application is structured.

### [MODULES.md](MODULES.md)
Detailed breakdown of application modules including their purposes, key files, dependencies, exports, and usage examples. Covers:
- App and Main modules
- Portfolio Context (state management)
- Components module
- Utils module (calculations and file parsing)
- Types module

### [FILES.md](FILES.md)
Complete file structure reference showing directory hierarchy, file purposes, and key file relationships. Includes:
- Root configuration files
- Source code organization
- Build and test outputs
- Import/export patterns
- File naming conventions

## How to Use These Codemaps

1. **For Understanding the System**: Start with ARCHITECTURE.md to get a high-level view
2. **For Development Work**: Refer to MODULES.md when working on specific features
3. **For Navigation**: Use FILES.md to locate specific files or understand project structure
4. **For Onboarding**: New developers should review all three documents in order

## Update Frequency

These codemaps should be updated when:
- Adding new major features or modules
- Making significant architectural changes
- Changing public APIs or interfaces
- Modifying the project structure substantially

Minor changes like bug fixes or small refactorings typically don't require codemap updates unless they affect the overall structure.
